## 先看成果：雷蒙的 `/morning` 指令

每天早上，我打開 Claude Code，只要輸入一個指令：

```
/morning
```

接下來 AI 就會自動幫我做完這些事：

1. 📧 **掃 Gmail**：列出今天的重要信件，標記優先級，合作信自動分析
2. 📅 **查 Google Calendar**：今天有什麼會議、什麼行程
3. ✅ **查 Notion 任務庫**：哪些待辦到期、哪些在進行中
4. 💬 **讀 Discord 訊息**：昨天社群裡聊了什麼、有沒有要回的
5. 📱 **抓 Facebook 貼文**：我昨天發的貼文互動狀況
6. 📝 **抓 WordPress 新文章**：有沒有新發布的內容要追蹤
7. 📓 **生成日記初稿**：根據以上資料，幫我寫昨日回顧
8. 🎯 **盤點今日任務**：問我今天的 Top 3 要做什麼

**全部自動跑完，我只要看報告、回答幾個問題就好。**

這不是什麼高深技術。它的核心其實就是兩件事：

> **第一，把你常用的工具授權給 AI。**
> **第二，把流程寫成一個指令。**

## 核心觀念：授權工具 → 組合流程

你在 2-2 學到 Claude Code 有 skills、commands、hooks 這些功能。`/morning` 就是一個 command（快捷指令），但它之所以能做這麼多事，是因為背後接了一堆工具：

| 工具 | 怎麼接上的 | AI 拿來做什麼 |
|:--|:--|:--|
| Gmail | MCP（Google OAuth 授權） | 掃收件匣、分類信件 |
| Google Calendar | MCP（Google OAuth 授權） | 查今日行程 |
| Notion | MCP（API Token） | 查任務庫待辦事項 |
| Discord | API（Bot Token） | 讀社群訊息 |
| Facebook | API（自有帳號抓取） | 抓貼文互動數據 |
| WordPress | API（REST API） | 抓最新發布的文章 |

**你不需要全部都接。** 從你最常用的 2-3 個工具開始就好。

> [!TIP]
> 還記得 2-2 提到的四條路線嗎？MCP、API、CLI、Browser Use，每個工具用最適合的方式連上就好。大部分常用工具都有 MCP，最簡單。

## 你也可以組出自己的工作流

`/morning` 是我的版本，但每個人的工作不同。你可以想想：

**你每天早上（或每天固定時間）都會做哪些事？**

例如：
- 老師：查學生繳交狀況 + 今天的課表 + 備課提醒
- 自由工作者：查信 + 查行事曆 + 今天的接案進度
- 內容創作者：查社群互動 + 查留言 + 今天要發什麼

只要把這些步驟寫成一個流程，讓 AI 來跑就好。

## 從手動到自動：進化三階段

我的 `/morning` 也不是一開始就這麼完整的。它經歷了三個階段：

### 階段一：每次手動說一遍

一開始我每天早上都跟 AI 說：「幫我看一下信、查一下行程、看 Notion 有什麼待辦⋯⋯」每天打一樣的話，打了兩個禮拜。

### 階段二：寫成指令，一鍵執行

後來我把這段流程寫成一個 Skill 或 command，存成 `/morning`。從此只要打兩個字就搞定。

### 階段三：AI 建議我自動化

用了幾週之後，Claude Code 每週會給我一份 **Insights 報告**（使用分析）。有一次它跟我說：

> 「你每天早上 9 點都會打 `/morning`。要不要我設定成 CronJob，每天自動跑？」

我說好。從此連打指令都不用了，每天早上 9 點 AI 自動執行，我打開電腦就能看到報告。

> [!TIP]
> **CronJob** 就是「排程任務」，你告訴電腦「每天幾點做什麼事」，它就會準時自動執行。就像鬧鐘一樣，設一次，天天叫你。Claude Code 內建這個功能，不用額外安裝。

## AI 不是萬能的：能做什麼、不能做什麼

裝了 MCP 不代表 AI 會自己亂來。幾個重要的邊界：

### ✅ AI 能做的
- 讀取你的信件、行程、Notion 頁面
- 搜尋和整理資料
- 建立草稿（Email 草稿、行程草稿）
- 在你確認後執行操作

### ❌ AI 不會自己做的
- **不會自動發送 Email**（只建草稿，你自己按發送）
- **不會自動刪除行程**（會先問你確認）
- **不會讀你沒授權的東西**（每個工具都需要你明確授權）

> [!WARNING]
> **Google 授權的安全提醒**
> 第一次連接 Gmail 或 Google Calendar 時，Google 會跳出授權頁面。這是正常的，你是在告訴 Google：「我允許 Claude Code 讀取我的信件/行事曆。」你隨時可以到 [Google 帳號設定](https://myaccount.google.com/permissions) 撤銷授權。

## 常見問題

### 「我不是工程師，也能組出自己的工作流嗎？」

可以。你不需要寫程式碼。最簡單的方式是跟 AI 說：

> 「我每天早上都會做這三件事：查信、看行程、確認今天的待辦。幫我把這個流程存成一個指令。」

AI 會幫你建好 command 檔案。之後你只要打 `/你取的名字` 就能執行。

### 「MCP 跟 API 有什麼差別？我要學哪個？」

MCP 是「幫你包好的 API」。對非工程師來說，學 MCP 就夠了。如果遇到沒有 MCP 的工具，再考慮用 API，到時候也是 AI 幫你串接，你只要告訴它你想做什麼。

### 「CronJob / Schedule 怎麼設定？」

直接跟 AI 說：「幫我設一個排程，每天早上九點自動跑 /morning。」它會幫你搞定。如果你想了解更多排程的細節，應用篇會有更深入的教學。
