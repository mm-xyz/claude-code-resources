# 把每日反思變成自動化 Skill：每天 3 分鐘，AI 幫你寫學習日記

> 迷你課第 3-3 單元｜應用篇
> 把 2-3 教過的反思方法論，做成一個每天能觸發的 `/journal` Skill。你跟 AI 聊 5 句話，當天的學習日記就寫好了，連隔天待辦都會幫你顧。

---

## 你可能遇過這個問題

你知道反思很重要，也知道寫日記、紀錄的重要性。聽過「事實 → 感受 → 反思 → 改進 → 行動」這條紀錄的路徑方法。但每天倒了晚上，真的要你去打開一個新檔案、整理今日的行動、任務、組織語言，你又放棄了。

**日記 OR 反思的門檻不是「不想做」，是「做的成本太高」**。

這篇就是把那個成本降到最低的：你不用打開任何檔案、不用想格式、不用組織語言。你只要跟 Claude Code 說一句「寫日記」，它會一題一題問你，你用嘴巴回答，它幫你寫好存起來。

---

## 核心觀念：為什麼要做成 Skill，而不是 prompt？

你當然可以每天複製貼上一段 prompt 叫 AI 幫你寫日記。但這有三個問題：

1. **沒有觸發詞，常常忘記放哪**：你要每次手動找 prompt、複製、貼上。摩擦一多就放棄。
2. **prompt 無法自動更新：今天用版本 A、想更新卻要手動很麻煩
3. **AI 不知道昨天的狀態**：昨天寫了什麼、列了什麼待辦，今天的對話完全忘掉。

**Skill 把這三個問題一次解決**：

- **Skill 是常駐的**：裝一次，之後永遠在 `/journal` 等你
- **Skill 有觸發詞**：你說「寫日記」「今天結束了」「journal」都會觸發，不用記指令
- **Skill 有記憶**：它被設計成每天開對話會自動讀前一天的日記，提醒你昨天列的待辦
- **Skill 可以進化**：用一週後覺得 Q3 問得太僵硬？直接請 CC 幫你改，AI 會自動把你的需求、新的方法更新到 `SKILL.md` 裡面，讓你不用自己去更新 SOP。

※ Skill 的本質是，讓你不用像以前 GPTs 和 Gems，要去找特定的對話機器人，你只要把需求丟給 CC，它會自動找合適的 Skill（技能），把你曾訓練它的方法和工具，拿來使用。

## 這個 Skill 的設計邏輯（5 個問題的順序）

好的反思日記，順序應是：**事實 → 感受 → 反思 → 改進 → 行動**：

| # | 問題 | 設計目的 | 背後的思考 |
|:--|:-----|:---------|:-----------|
| Q1 | 今天做了哪些事？發生了什麼？ | **記錄事實** | 先從最簡單的開始，降低回答壓力。「發生了什麼」比「你學到什麼」容易回答 |
| Q2 | 有什麼讓你驚喜、覺得有趣的時刻？ | **捕捉正面體驗** | 刻意引導你注意正面的學習體驗。人容易記住挫折，但驚喜時刻才是持續學習的動力 |
| Q3 | 哪些做得不錯？哪些可以更好？有卡住的地方嗎？ | **自我反思** | 先問「做得不錯」再問「可以更好」，避免只聚焦在問題上。卡關紀錄是最有價值的學習資產，未來遇到同樣問題可以回來翻 |
| Q4 | 明天我「能控制」並改進的一件小事是什麼？ | **聚焦可控的改變** | 關鍵字是「能控制」和「一件小事」。不是「我要變更厲害」這種空泛目標，而是「下次先問 AI 再自己查」這種具體行為。小改變累積起來才是真正的成長 |
| Q5 | 明天最重要的三件事？ | **待辦規劃** | 限制「三件」避免列出一長串做不完的清單。用 `- [ ]` checkbox 格式，隔天 AI 會自動提醒，形成閉環 |

### 為什麼是 5 題，不是 3 題或 10 題？

- **3 題太少**：少了反思和改進的環節，變成流水帳
- **10 題太多**：寫日記變成負擔，第三天就不寫了
- **5 題剛好**：涵蓋完整的反思循環，但回答起來只要 3 分鐘

### 為什麼逐題問、不一次丟？

一次丟 5 題看起來像問卷，你會敷衍或直接跳過。**逐題問像聊天**，每次只要想一件事，心理負擔小很多。

### 待辦的閉環設計

不用獨立的 todo.md，待辦直接寫在日記的「明天要做的事」裡。

閉環運作：

1. 今天寫日記 → Q5 列出明天的三件事（`- [ ]` 格式）
2. 明天開對話 → AI 讀前一天日記 → **主動口頭提醒**你昨天列了哪些待辦
3. 明天寫日記 → Q1 回顧有沒有做到 → 新的 Q5 列出後天的事

這樣你不需要學任何 todo 工具，**日記本身就是任務追蹤系統**。

## 動手做

### Step 1：安裝 `learning-journal` Skill

> [!IMPORTANT]
> **升級包**
> - 🔒 Skill `learning-journal`（完整 Skill 資料夾在 `skills/learning-journal/`）
> - 把 `skills/learning-journal/SKILL.md` 整份內容貼給 Claude Code，說「幫我把這個 Skill 裝到 `.claude/skills/journal/`」，AI 會自動建立資料夾、寫入檔案、驗證安裝。

或者你如果跑過 pro-kit 01「AI 分身起始助手」，直接告訴 Claude Code：

```
幫我把 skills/learning-journal/ 複製到 000_Agent/skills/journal/
```

### Step 2：驗證安裝

重開一個 Claude Code 對話，看 `/` 選單有沒有 `/journal`。

如果沒有，三個檢查：

1. `ls -la ~/.claude/skills/journal/SKILL.md` 檔案存在嗎？
2. `head -10 ~/.claude/skills/journal/SKILL.md` 看 `name:` 是否等於資料夾名 `journal`
3. 重啟 Claude Code（偶爾 watcher 沒偵測到新目錄）

### Step 3：試寫第一篇

跟 Claude Code 說：

```
寫日記
```

或

```
今天結束了
```

AI 會依序問你 5 個問題，你口語回答就好。答完它會把整篇生成出來給你確認，存到 `journals/YYYY-MM-DD.md`。

### Step 4：隔天驗證閉環

隔天開一個新的 Claude Code 對話。如果昨天寫了日記且有「明天要做的事」，AI 會在對話一開始主動跟你說：

> 早安！昨天你列了這些今天想做的事：
> - [ ] xxxxx
> - [ ] xxxxx
>
> 要從哪個開始？

這就是閉環成功的訊號，你的日記、待辦、提醒全部串起來了，只靠一個 Skill。

## 📍 【升級玩法】搭配 2-3 的組合技

還記得 2-3 嗎？雷蒙在那一篇攤開了自己的 `/morning` 真實工作流——**為什麼我做起來這麼低摩擦力？不是因為有什麼魔法，而是我把日常會留紀錄的工具（Gmail / Calendar / Notion / Discord / Git...）都授權給 AI Agent 去存取**。AI 不用問「你今天幹嘛」，它看資料就知道了。

所以兩篇的關係是這樣：

- **2-3**：雷蒙的真實案例——把工具授權給 AI，讓 AI 主動知道你的一天發生什麼事（這是**基礎建設**）
- **3-3**：把反思流程封裝成 `/journal` Skill，配合基礎建設，每天 3 分鐘就搞定一天的日記（這是**落地工具**）

### 從「靠你回想」到「AI 比你還清楚你今天做了什麼」

上面那個 5 題版本的 `/journal` 是**入門版**——假設 AI 不知道你今天做了什麼，所以用問的讓你回想。適合剛開始、還沒把工具授權給 AI 的你。

但一旦你照著 2-3 的做法，把 Gmail / Calendar / Notion / Discord / Git commit 這些日常會留紀錄的地方都授權給 AI，這個 Skill 會整個**質變**：AI 不用再問「你今天做了什麼」，它**已經知道了**。

這時候 `/journal` 的角色會整個質變：

- 入門版：「**靠你回想**，幫你整理成日記」
- 升級版：「**根據真實數據**，挖出你自己沒注意到的觀感、思考、心情」

舉幾個雷蒙真實遇過的例子：

- 今天發了 3 篇社群貼文 → AI 會問「為什麼下午那篇你刻意只貼 Threads 沒貼 FB？是覺得這個觀點還沒準備好給廣泛的觀眾嗎？」
- Calendar 顯示連續 3 個會議都是同一個學員 → AI 會問「這位學員最近佔了你不少時間，你有觀察到什麼共通的卡點嗎？」
- Git log 顯示今天反覆 commit 同一個檔案 5 次 → AI 會問「這段卡了一陣子，是技術問題、還是設計判斷在拉扯？」

**這就是我最享受、最過癮的部分**：你的腦袋可以清空，不用再費力記今天到底做了什麼，AI 替你記、替你跑。你的角色變成「生活體驗的主人」——當 AI 提出深度思考時，**回答出 AI 抓不到的那些蛛絲馬跡**：直覺、感受、那個你還沒講出口的念頭。

**這才是 AI Agent 真正的價值**：不只幫我們省時間，更讓你看到自己平常看不到的自己。

> [!TIP]
> **怎麼開始升級？不用一次到位**
> 先把你「最常留紀錄的一個平台」授權給 AI——雷蒙建議從 Calendar 或 Gmail 開始（門檻最低、訊號最強）。跑一週看看 AI 問的問題會不會變得更深。
>
> 習慣了再加第二個、第三個。1-3 個月後，你的 `/journal` 會變成另一個層次的東西，你會回頭發現「天啊，原來我之前的反思這麼表面」。

## 常見問題 FAQ

**Q：我有些天真的什麼都不想寫，可以跳過嗎？**
A：可以。任何一題說「跳過」就跳過。Skill 不會逼你寫，只會在你願意寫的時候把門檻降到最低。重點是「寫了一半」比「完全沒寫」有價值。

**Q：我想改問題，例如加一題「今天最累的一刻」？**
A：打開 `~/.claude/skills/journal/SKILL.md`，在 Step 1 的問題列表加一題、同步更新 Step 2 的日記格式就好。這就是 Skill 的優勢：**你可以隨時迭代**，不用重裝。

**Q：日記寫出來我想推上 GitHub 備份，AI 會幫忙嗎？**
A：會。Step 4 會問你要不要推 GitHub，commit message 格式是 `journal: YYYY-MM-DD 學習日記`。

**Q：可以用在工作之外嗎？例如健身、閱讀？**
A：可以，但會建議你另建一個 Skill（例如 `fitness-journal`），用不同的 5 個問題。把 `journal` 專用在「Claude Code 學習」這個脈絡，效果最好，脈絡越清楚，AI 的回答品質越高。

**Q：寫一陣子後怎麼回顧？**
A：跟 Claude Code 說「幫我看一下上週的日記」或「整理我這個月學到的五件事」，它會自動讀 `journals/` 資料夾彙整給你。這是純文字 Markdown 的好處，**未來任何 AI 都能讀**。
