# 學習日記

用 5 個引導問題幫學員寫下每天的學習紀錄。
問題設計依照「事實 → 感受 → 反思 → 改進 → 行動」的順序，引導學員從回顧走到規劃。

## 觸發條件

學員說以下任何一句話時觸發：
- 「寫日記」
- 「今天結束了」
- 「學習紀錄」
- 「journal」
- 「記錄一下今天」

## 日記檔案位置

`journals/YYYY-MM-DD.md`

例如：`journals/2026-04-15.md`

如果 `journals/` 資料夾不存在，自動建立。

## 寫日記流程

### Step 0：確認日期與檢查既有日記

1. 先執行 `date` 確認今天的日期
2. 檢查 `journals/YYYY-MM-DD.md` 是否已經存在
   - **已存在**：告知學員「今天的日記已經寫過了」，詢問要「補充內容」還是「重新寫」
     - 補充內容：讀取現有日記，跳到 Step 1 讓學員補充，最後合併寫回
     - 重新寫：從 Step 1 重新開始，覆蓋原檔案
   - **不存在**：繼續下一步

3. 讀取前一天的日記（如果存在）。
如果前一天的「明天要做的事」有內容，先口頭提醒學員：

> 昨天你列了這些想做的事：
> - [ ] xxxxxx
> - [ ] xxxxxx
>
> 等等寫日記時可以回顧一下有沒有做到。

### Step 1：引導問答（逐題問，不要一次全丟）

用 AskUserQuestion 工具逐題引導，每題等學員回答後再問下一題。
學員說「跳過」就跳過該題。

**Q1：今天做了哪些事？發生了什麼？**
> 不管大小都算——裝了什麼、試了什麼、改了什麼、讀了什麼都可以說。

**Q2：有什麼讓你驚喜、覺得有趣的時刻？**
> 可能是 AI 做了一件讓你意想不到的事，或是你突然搞懂了一個觀念。

**Q3：哪些做得不錯？哪些可以更好？有卡住的地方嗎？**
> 做得好的值得記下來，卡住的也是。還沒解決沒關係，記錄下來之後可以回來看。

**Q4：明天我「能控制」並改進的一件小事是什麼？**
> 不用想很大的改變，一件小事就好。例如「下次先問 AI 再自己查」或「每做完一段就推上 GitHub」。

**Q5：明天最重要的三件事？**
> 不用想太多，列出來就好。沒想到可以跳過。

### Step 2：生成日記

根據學員的回答，生成日記檔案。格式如下：

```
# YYYY-MM-DD 學習日記

## 今天做了什麼
（根據 Q1 回答，用條列式整理，保留學員的原話和細節，不要摘要）

## 驚喜時刻
（根據 Q2 回答，保留學員的語氣）

## 反思
（根據 Q3 回答。做得好的和可以更好的都列出來。如果有卡關，用「問題 → 解法」格式）

## 明天的一個小改進
（根據 Q4 回答，一句話就好）

## 明天要做的事
- [ ] 第一件事
- [ ] 第二件事
- [ ] 第三件事
```

規則：
- 保留學員的原話和所有細節，不要摘要或簡化
- 跳過的題目不要留空標題，直接省略該區塊
- 「明天要做的事」用 `- [ ]` checkbox 格式

### Step 3：確認與存檔

把生成的日記內容展示給學員看，問：

> 日記寫好了，有要修改的地方嗎？沒問題的話我就存檔了。

學員確認後存檔到 `journals/YYYY-MM-DD.md`。

### Step 4：推上 GitHub

存檔後問學員：

> 要幫你推上 GitHub 嗎？養成習慣，每天存檔一次最安全。

學員同意就執行 git add + commit + push。
commit message 格式：`journal: YYYY-MM-DD 學習日記`

## 隔天開對話時的行為

當學員開啟新對話時，如果前一天的日記存在且有「明天要做的事」，主動口頭提醒：

> 早安！昨天你列了這些今天想做的事：
> - [ ] xxxxxx
> - [ ] xxxxxx
> - [ ] xxxxxx
>
> 要從哪個開始？還是有新的計畫？

如果前一天有「明天的一個小改進」，也一起提醒：

> 還有，你昨天說今天想改進的一件事是：「xxxxxx」

## 注意事項

- 問問題時語氣親切自然，像朋友聊天，不要像考試
- 不要催促學員，每個人的學習節奏不同
- 如果學員只寫了很短的回答，不要追問，尊重他們的表達方式
- 日記是私人的學習紀錄，不需要寫得很正式
