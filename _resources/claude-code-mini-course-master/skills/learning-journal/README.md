## 你可能遇過這個問題

你開始用 Claude Code 了，每天都在試新東西——今天裝了 MCP、明天做了一個小工具、後天把 CLAUDE.md 改了三次。

但一週後回頭看，你記不清自己學了什麼、卡在哪裡、又是怎麼解決的。那些「啊原來這樣做就行了！」的瞬間，全部蒸發了。

你知道應該做筆記，但每次想到「要打開一個檔案、想格式、寫內容」就算了。

**學習日記 Skill 專治這個問題。** AI 問你 5 個問題，你口語回答就好，它幫你整理成漂亮的日記存起來。每天 3 分鐘。

## 裝完之後你會得到什麼

- 一個 `/journal` Skill，說「寫日記」或「今天結束了」就會觸發
- AI 用 5 個問題引導你回顧，不用自己想該寫什麼
- 回答完自動生成當天的學習日記（Markdown 格式）
- 隔天開對話，AI 會自動讀前一天日記，提醒你昨天列的待辦
- 所有日記存在 `journals/` 資料夾，隨時可以回顧

## 怎麼安裝？

### 方式 1：直接複製資料夾（推薦）

把這個資料夾複製到你的 Skill 安裝位置，並改名為 `journal`（對應 `SKILL.md` 裡的 `name: journal`）：

```bash
# 如果你跑過 pro-kit 01「AI 分身起始助手」
cp -r learning-journal/ 000_Agent/skills/journal/

# 或者用 Claude Code 預設位置
cp -r learning-journal/ ~/.claude/skills/journal/
```

### 方式 2：貼給 Claude Code 代裝

把 `SKILL.md` 整份內容貼給你的 Claude Code，跟它說：

> 幫我把這個 Skill 裝到 `.claude/skills/journal/`

AI 會自動建立資料夾、寫入檔案、驗證安裝。

### 驗證

重開一個 Claude Code 對話，打 `/journal` 或跟它說「寫日記」，應該會觸發 5 題引導問答。

## 怎麼用？

```
寫日記
```

或

```
/journal
```

或

```
今天結束了
```

AI 會用 5 個問題引導你：做了什麼 → 驚喜時刻 → 反思 → 小改進 → 明天三件事。回答完自動存到 `journals/YYYY-MM-DD.md`。

## 5 題反思框架的設計邏輯

這 5 題不是隨便排的，順序是「事實 → 感受 → 反思 → 改進 → 行動」，經過驗證的反思循環：

| # | 問題 | 設計目的 |
|:--|:-----|:---------|
| Q1 | 今天做了哪些事？發生了什麼？ | 記錄事實（降低起步壓力）|
| Q2 | 有什麼讓你驚喜、覺得有趣的時刻？ | 捕捉正面體驗（持續動力）|
| Q3 | 哪些做得不錯？哪些可以更好？有卡住的地方嗎？ | 自我反思（卡關紀錄最值錢）|
| Q4 | 明天我「能控制」並改進的一件小事是什麼？ | 聚焦可控的改變（小改變累積成長）|
| Q5 | 明天最重要的三件事？ | 待辦規劃（限制三件避免 overwhelm）|

**為什麼 5 題、不是 3 或 10？** 3 題太少（沒有反思環節）、10 題太多（學員第三天就不寫了）、5 題剛好涵蓋完整循環又只要 3 分鐘。

**為什麼逐題問、不一次丟？** 一次丟 5 題像問卷會被敷衍；逐題像聊天，心理負擔小。

**待辦閉環**：Q5 列出明天三件事 → 隔天開對話 AI 自動提醒 → 新日記 Q1 回顧有沒有做到。學員不用學額外的 todo 工具，日記本身就是任務追蹤系統。
